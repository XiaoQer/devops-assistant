# Project 详情页设置面板布局

## 背景

Project 详情页当前用分类卡片网格展示基础信息、Owner、GitHub 和 Aliyun 元信息。
该布局在宽屏下出现左右高度不齐、空白区域过大、Aliyun 横跨整行导致节奏割裂的问题，
不适合后台管理类详情配置页。

## 范围

- 将 Project 详情页分类卡片网格改为单列设置面板。
- 每个分类以一条横向设置行呈现：左侧分类说明，中间字段展示/编辑，右侧操作。
- 保留基础信息、Owner、GitHub、Aliyun 四个分类。
- 保留每个分类独立进入编辑态、保存、取消能力。
- 保持字段展示态为只读，不直接展示表单控件。

## 非目标

- 不修改 Project 列表页。
- 不修改后端 API、数据库字段或保存 payload。
- 不改变 Kubernetes 和 Registry 独立菜单。
- 不新增 Project 治理字段。

## 验收条件

- Project 详情页不再使用分散的两列分类卡片网格。
- 四个分类在一个统一设置面板中按行排列，视觉节奏一致。
- 每行字段信息密度适合后台管理场景，避免大面积空白。
- 每个分类仍可分别进入编辑态并保存。
- 前端构建和仓库验证通过。
