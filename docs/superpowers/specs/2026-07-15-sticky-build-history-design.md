# 构建历史栏吸顶布局设计

## 背景

构建详情日志通常远高于左侧历史列表。双列 Grid 以顶部对齐后，历史列表结束位置以下会出现贯穿
页面的大块空白，削弱详情区域的视觉连续性。

## 设计

- 桌面端左侧 `ApplicationBuildHistory` 在构建浏览页中吸顶，顶部位置避开全局导航并保留小间距。
- 历史栏高度受当前视口约束，列表在卡片内部滚动；右侧详情继续使用页面滚动。
- 历史栏自身不拉伸到右侧日志高度，避免用空卡片填充无意义空间。
- 窄屏（不超过 760px）取消吸顶，恢复历史列表在详情上方的普通文档流，并保持现有 280px 列表上限。
- 不改动构建选择、路由、轮询、步骤日志或 API 行为。

## 验证

- 用纯状态/样式契约检查确认桌面吸顶高度与移动端复位规则存在。
- 运行前端测试、类型检查、生产构建和 `./scripts/verify.sh`。

## 非目标

- 不改变左右列宽。
- 不折叠或隐藏构建历史。
- 不为左右两列增加联动滚动。
