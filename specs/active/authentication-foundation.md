# 功能：身份认证基础

- 状态：草稿
- 负责人：shaoqian.li
- 创建日期：2026-07-03

## 背景

Aegis 当前没有真实身份认证。后端通过可由调用方伪造的 `X-User` 请求头记录操作人，
前端还硬编码了演示用户。这使受保护操作和后续配置审计都无法依赖可信的当前用户。

本需求为预置管理员提供登录、数据库会话、当前用户查询和退出能力，并统一保护现有
API。它是后续“环境配置审计时间线”的前置能力。

## 当前行为

- `backend/app/__init__.py` 注册全部业务 Blueprint，但没有认证守卫。
- 部分 Route 从 `X-User` 请求头读取操作人，缺省为 `local-user`。
- `frontend/src/api/client.ts` 没有身份、Cookie、CSRF 或 401 处理。
- `frontend/src/router/index.ts` 没有登录页面和路由守卫。
- `frontend/src/layouts/PortalLayout.vue` 硬编码用户姓名。
- 仓库没有用户、会话模型或相关 Alembic migration。

## 目标行为

预置管理员可使用用户名和密码登录。服务端创建可过期、可撤销的数据库会话，并通过
HttpOnly Cookie 识别后续请求。除健康检查和登录外，所有 API 均要求有效会话。
前端能够恢复当前用户、保护页面路由、处理会话失效并完成退出登录。

认证成功后，业务 Route 和 Service 从服务端验证过的当前用户获取操作人，不再信任
调用方提供的 `X-User`。

## 范围

- 包含：`User` 和 `UserSession` 持久化模型及 Alembic migration。
- 包含：密码安全哈希、随机会话令牌、令牌摘要存储、过期和撤销。
- 包含：登录、当前用户和退出登录 API。
- 包含：全局 API 认证守卫和非只读请求的 CSRF 校验。
- 包含：非交互的管理员初始化命令；凭据从环境变量读取。
- 包含：登录页、认证状态 Store、前端路由守卫、401 处理和当前用户展示。
- 包含：将现有 `X-User` 操作人来源迁移为已认证用户。

## 非目标

- 不包含：公开注册、邀请、找回密码或修改密码。
- 不包含：角色授权、项目级权限或成员角色强制执行。
- 不包含：SSO、OAuth、OIDC、多因素认证或 API Key。
- 不包含：用户和会话管理界面。
- 不包含：配置审计时间线或历史版本恢复。
- 不包含：分布式登录限流；在补充网关或共享存储限流前，不宣称认证能力生产就绪。

## 验收条件

- [ ] 给定有效的预置管理员凭据，当用户登录时，则服务端创建数据库会话、设置
      HttpOnly Cookie，并通过统一 API 响应返回当前用户和 CSRF Token。
- [ ] 给定错误用户名、错误密码或禁用账号，当用户登录时，则响应使用相同的通用
      错误消息，且不暴露账号是否存在或是否禁用。
- [ ] 给定有效会话，当客户端查询当前用户时，则返回用户 ID、用户名和显示名，但
      不返回密码哈希、会话令牌或令牌摘要。
- [ ] 给定有效会话，当用户退出时，则当前数据库会话被撤销、Cookie 被清除，原会话
      不能继续访问受保护 API。
- [ ] 给定缺失、过期、已撤销或属于禁用用户的会话，当访问受保护 API 时，则返回
      `401` 和 `AUTHENTICATION_REQUIRED`。
- [ ] 给定有效会话但缺失或错误的 CSRF Token，当发起非只读请求时，则返回 `403`
      和稳定的错误代码；安全的只读请求不要求 CSRF Token。
- [ ] 除健康检查和登录外，所有现有 API 都受认证守卫保护。
- [ ] 现有部署、审批和配置等业务操作使用已认证用户身份，不再信任 `X-User`。
- [ ] 前端未登录访问业务页面时跳转登录页并保留原目标；登录后返回原页面，刷新后
      可恢复会话，401 和退出登录均回到登录页。
- [ ] 管理员初始化命令保持非交互，凭据不会写入代码、migration、日志或测试数据。
- [ ] 相关后端自动化检查覆盖密码哈希、认证端点、会话状态、全局保护、CSRF 和统一
      响应结构；前端类型检查和生产构建通过。
- [ ] `./scripts/verify.sh` 通过，`docs/current-state.md` 准确记录已实现能力和已知缺口。

## 设计说明

### 数据模型

`User` 保存唯一用户名、显示名、密码哈希、启用状态及创建/更新时间。密码仅使用
Werkzeug 提供的安全密码哈希接口处理，不存储或返回明文。

`UserSession` 保存用户外键、随机会话令牌的 SHA-256 摘要、CSRF Token 摘要、
过期时间、撤销时间和创建时间。原始会话令牌只存在于浏览器 Cookie 中。默认会话时长
为 8 小时，并允许通过应用配置调整。

Migration 只创建数据结构。管理员由非交互 CLI 命令从环境变量读取用户名、显示名和
密码后创建或显式失败，不在 migration 中植入固定凭据。

### HTTP 与安全边界

- `POST /api/auth/login` 校验输入和凭据，创建会话并设置 Cookie。
- `GET /api/auth/me` 返回当前用户及供前端使用的 CSRF Token。
- `POST /api/auth/logout` 撤销当前会话并清除 Cookie。
- Cookie 使用 `HttpOnly` 和 `SameSite=Lax`；生产配置启用 `Secure`。
- `POST`、`PUT`、`PATCH` 和 `DELETE` 请求必须通过请求头提交 CSRF Token。
- 未认证返回 `401 / AUTHENTICATION_REQUIRED`，CSRF 失败返回 `403` 和稳定错误码，
  JSON 输入校验失败返回 `400`。
- 认证失败使用通用消息。日志不得包含密码、Cookie、原始会话令牌或 CSRF Token。
- CORS 必须允许受控前端来源携带凭据，不再使用无约束的默认配置。

### 后端组件

Route 只负责输入、Cookie 和响应处理。认证、密码验证、会话创建/撤销和当前用户解析
由认证 Service 完成。全局请求钩子验证会话并把用户放入 Flask 请求上下文，现有业务
Route 从该上下文获得可信操作人。

健康检查和登录通过明确的端点白名单跳过会话验证。其他 Blueprint 默认受保护，避免
新增 Route 因忘记添加装饰器而意外公开。

### 前端组件

认证 Store 保存当前用户、CSRF Token 和初始化状态，不持久化会话令牌。API 客户端
携带 Cookie，为非只读请求附加 CSRF Token，并在 401 时清除身份状态。

登录页接受用户名和密码。路由守卫在认证初始化完成后决定放行或跳转，并使用受控的
站内路径保存原目标，避免开放重定向。各 Layout 通过统一用户组件展示当前用户和退出
操作，不再硬编码演示身份。

### 备选方案

- Flask 签名 Cookie 会话实现更少，但难以主动撤销单个会话，不选。
- JWT 更适合跨服务或多类客户端；当前同源 Vue + Flask 架构不需要其刷新和撤销
  复杂度，不选。
- 数据库会话便于撤销、过期、后续审计和未来会话管理，因此采用。
