# 产品定义

## Aegis 是什么

Aegis 是面向容器化应用、以项目为中心的软件交付控制面。它在统一工作空间中连接
代码仓库、构建流水线、镜像仓库、Kubernetes 环境、发布、审批和运行态检查。

当前产品处于工程 MVP 阶段，适合迭代开发和演示，尚不是生产就绪的多租户平台。

## 用户与任务

- 开发者接入仓库、配置环境、发布、排查失败并执行回滚。
- 项目负责人管理项目内的应用、成员、集群和镜像仓库。
- 交付审批人批准或拒绝受治理的发布。
- 平台工程师提供 Kubernetes、Tekton、镜像仓库和访问基础设施。

## 产品模型

`Project` 是顶层治理和导航边界，限定应用、成员、Kubernetes 集群和镜像仓库范围。

`Application` 是项目内的核心交付对象。环境、配置、流水线执行、发布、审批和运行
状态共同构成应用交付工作空间。

## 核心价值链路

```text
Git 代码仓库
  → 仓库分析
  → 应用与环境
  → 部署计划
  → 必要时审批
  → Tekton 流水线与 Kaniko 镜像构建
  → Kubernetes 部署
  → 发布历史与运行态检查
```

## 产品原则

- 围绕项目和应用组织体验，而不是围绕原始基础设施对象。
- 优先展示决策、交付进度和下一步动作，再展示底层细节。
- 风险操作必须可解释、可确认、可审计。
- AI 用于理解平台意图和辅助工作流，而不是充当通用聊天机器人。
- 业务历史不依赖短暂存在的执行资源。

## 当前边界

Aegis 当前不定位为：

- 完整的可观测性或事件管理平台；
- 完整的企业身份、租户隔离或权限系统；
- Service Mesh 或渐进式交付控制器；
- 自主运行的 AI 运维 Agent；
- Kubernetes、Tekton、镜像仓库或源码管理系统的替代品。
