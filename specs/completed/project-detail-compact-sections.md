# Project 详情页分类卡片紧凑化

## 背景

Project 详情页当前按基础信息、Owner、GitHub、Aliyun 分类展示治理元信息，但分类卡片
高度和留白偏大，页面在普通视口中需要较多滚动，不符合后台管理类配置页的信息密度。

## 范围

- 保持 Project 详情页现有信息类别和分类编辑能力。
- 压缩分类卡片头部、字段行、图标和按钮尺寸。
- 将分类区域调整为更紧凑的后台配置面板风格。
- 保持 Project Name 和 Project Key 只读。
- 保持 Kubernetes 和 Registry 继续走独立菜单。

## 非目标

- 不修改 Project 列表页 `/project-center/projects`。
- 不修改后端 API 或数据库字段。
- 不新增 Project 详情页信息类别。
- 不改变保存逻辑和分类编辑边界。

## 验收条件

- Project 详情页分类卡片高度明显缩小，减少大面积空白。
- 基础信息、Owner、GitHub、Aliyun 仍可分别进入编辑态并保存。
- 字段在展示态为只读展示，不直接显示为表单。
- 页面在中等视口下仍保持两列或合理紧凑布局，小屏幕下可单列展示。
- 前端生产构建和仓库验证脚本通过。
