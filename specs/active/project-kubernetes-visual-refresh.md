# 功能：Project Kubernetes 页面视觉优化

- 状态：已确认，待实现
- 负责人：Aegis Agent
- 创建日期：2026-07-13

## 背景

Project Kubernetes 页面已具备集群接入、环境标签、连接测试和集群管理能力，但当前页面
在宽屏下纵向留白较大，单个集群占用整行大卡片，状态、连接信息和操作缺少明确层级。
添加入口与表单弹窗也需要和新的资源卡片视觉语言统一。

## 当前行为

- 页面头部标题尺寸和上下留白较大，主要内容进入视口较晚。
- 集群列表使用单列全宽卡片，少量字段横向分散，多个操作按钮并列堆叠。
- API Server、最近测试时间、状态标签和治理信息处于近似层级。
- 添加路径选择和表单弹窗已可用，但表单仍是单一长区块，kubeconfig 与普通文本字段缺少
  视觉区分。

## 目标行为

页面采用双列资源卡片方向：在保持 Project Center 浅色后台风格的前提下，提高首屏信息
密度、状态可读性和操作层级。添加入口与表单弹窗使用同一套边框、圆角、间距和语义颜色，
但不改变任何 API、数据模型或业务流程。

## 范围

- 包含：压缩页面头部的纵向留白和标题尺度，保持主操作清晰可见。
- 包含：集群列表在宽屏使用双列资源卡片，窄屏自动切换为单列。
- 包含：重组卡片头部、API Server 信息区、状态徽标、最近测试时间和操作区。
- 包含：“测试连接”作为主要操作，“编辑”作为次要操作；“设为默认”和“删除”收纳到更多
  操作菜单，删除继续使用已有确认流程。
- 包含：优化新建路径选择卡片的可用/不可用状态。
- 包含：把集群表单拆分为“基本信息、连接配置、治理设置”三个视觉区块，kubeconfig 使用
  等宽字体输入区。
- 包含：响应式样式、前端类型检查、生产构建和完整仓库验证。

## 非目标

- 不包含：修改 API、数据库模型、集群业务规则或连通性测试逻辑。
- 不包含：新增筛选、搜索、排序、分页或详情路由。
- 不包含：实现 Aliyun ACK 初始化。
- 不包含：重做 Project Center 全局导航或其他资源页面。

- [ ] 最近测试时间位于卡片底部；测试连接和编辑直接可见；设为默认和删除位于更多操作菜单，

## 设计说明

- 只修改 `frontend/src/views/ProjectKubernetesSettings.vue` 的模板结构和局部样式；若现有
  `PageHeader` 无需全局修改，则不扩大到其他页面。
- 宽屏卡片网格使用 `repeat(2, minmax(0, 1fr))`，在约 1100px 内容宽度以下切换为单列。
- 更多操作使用 Element Plus Dropdown；删除沿用 `removeCluster()` 中的确认弹窗。
- 语义颜色继续使用 Project Center 的蓝色主操作、绿色连接成功、琥珀色失败/未测试和中性
  灰色辅助信息，不引入新的全局主题。
- 弹窗继续使用 `governance-form-dialog` 基座，页面局部样式只补充分区、选择卡和 kubeconfig
  输入的视觉规则。
