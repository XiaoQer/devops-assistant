# 功能：Portal 平台介绍与工作空间入口重构

- 状态：已验收
- 负责人：产品负责人 / Codex
- 创建日期：2026-07-02
- 验收日期：2026-07-03

## 背景

当前 Portal 首页以“Choose your workspace”为主标题，直接展示 Project Center 和
DevCenter 两张入口卡片。页面没有先解释 Aegis 是什么、解决什么问题以及核心交付
链路；“治理与交付相互独立”的描述也偏架构术语，不能形成清晰的产品第一印象。

## 当前行为

- 页面首屏标题是 “Choose your workspace”。
- 两张大卡片分别介绍 Project Center 和 DevCenter。
- 底部用一条边界关系说明连接两个中心。
- 页面视觉更像系统选择器，而不是产品门户。

## 目标行为

Portal 首屏首先将 Aegis 定义为 Software OS，说明平台如何把软件定义、团队协作、
配置、交付、治理和运行状态组织成统一上下文；工作空间入口作为下一层操作区域，
解释两个中心各自解决什么问题以及用户何时进入。页面应呈现为企业内部应用门户，
而不是商业官网或营销落地页。

## 范围

- 重构 `PortalHome.vue` 的信息层级、文案和视觉样式。
- 增加平台定位、核心价值和从代码到运行的交付链路介绍。
- 保留 Project Center 与 DevCenter 的入口。
- 保持桌面端与移动端响应式布局。

## 非目标

- 不改变 Project Center 和 DevCenter 的业务功能。
- 不改变后端 API、数据模型或权限体系。
- 不在本次需求中重构 Portal 顶部导航和用户身份信息。

## 验收条件

- [x] 首屏明确出现 Aegis 名称、平台定位和一段可理解的中文介绍。
- [x] 首屏明确表达 Software OS 定位及其统一软件上下文、全生命周期管理思想。
- [x] 页面表达 Git、Pipeline、Registry、Kubernetes、Release 等核心交付能力。
- [x] Project Center 与 DevCenter 的职责用用户任务语言描述。
- [x] 两个入口能够继续打开对应工作空间。
- [x] 1440px 桌面视口和移动视口下信息层级清晰、无横向溢出。
- [x] 前端类型检查和生产构建通过。
- [x] 浏览器中完成视觉检查。

## 设计说明

- 使用“平台介绍 → 交付链路 → 工作空间入口”的信息顺序。
- 避免把“上游系统/下游系统”和“边界”作为主要产品文案。
- 视觉风格延续现有蓝色设计系统，采用克制、稳定、信息清晰的企业应用布局。
- 不使用巨型营销口号、大面积渐变和宣传式承诺文案。
- 平台介绍应像内部系统的“平台概览”，并提供能力范围与明确入口。
- 正文字号和辅助文字对比度满足企业应用的持续阅读需求，不使用过小、过浅的文字。
