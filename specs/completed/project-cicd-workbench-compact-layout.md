# 功能：Project CI/CD 工作台紧凑布局

- 状态：已验收
- 负责人：Codex
- 创建日期：2026-07-14

## 背景

Project CI/CD 工作台已经能够聚合多个 Application 的构建与发布状态，但页面顶部、筛选区和
Application 卡片占用空间偏大，不利于同时浏览 10–30 个服务；页面底部的最近执行列表与卡片
详情入口重复，增加了视觉和数据加载负担。

## 当前行为

`frontend/src/views/PipelineRuns.vue` 使用通用大标题、独立筛选容器、带标题的 Application 外层
容器，并额外加载和展示最近 PipelineRun 列表。卡片为三列且内部预留固定高度。

## 目标行为

工作台首屏以紧凑标题、筛选工具栏和内容自适应的 Application 卡片为主；宽屏能展示更多卡片，
不再展示或请求最近执行列表，Pipeline 详情仍可从 Application 卡片进入。

## 范围

- 包含：压缩工作台标题、说明、操作和筛选区域的纵向空间。
- 包含：缩小卡片间距和内边距，移除固定留白并优化响应式列数。
- 包含：移除最近执行区域及其独立请求、分页和状态。

## 非目标

- 不包含：修改构建、发布、审批或重试的业务行为。
- 不包含：修改 Application 详情页中的构建版本和 Pipeline 记录。
- 不包含：修改后端 API。

## 验收条件

- [x] 标题、操作和筛选控件在紧凑区域内展示。
- [x] 卡片以 4/3/2/1 列响应式排列且高度由内容决定。
- [x] 卡片不为缺失的 Commit 信息预留固定空白。
- [x] 卡片的构建和 Application 详情使用带悬浮说明的紧凑图标按钮。
- [x] 自动刷新或筛选只加载工作台聚合数据，不再请求最近执行列表。
- [x] 构建、发布、重试、查看进度和 Application 详情入口继续可用。
- [x] 前端类型检查、生产构建和项目完整验证通过。
- [x] 面向用户和项目记忆的文档已更新。

## 设计说明

保留现有路由和卡片交互。使用页面专属紧凑标题样式，不修改共享 `PageHeader`。筛选栏并入
Application 内容容器顶部；移除 PipelineRun 列表相关状态和请求，但保留卡片跳转详情的能力。
