# 功能：构建历史栏吸顶布局

- 状态：已确认
- 负责人：Codex
- 创建日期：2026-07-15

## 背景

右侧长日志远高于左侧历史列表时，左列下方形成大块无内容白区。

## 当前行为

历史列表只设置内部最大高度，外层历史卡片仍随页面滚走，右侧详情继续向下延伸。

## 目标行为

桌面端历史栏固定在可视区域内并独立滚动，移动端保持普通上下布局。

## 范围

- 包含：桌面端吸顶、视口高度约束和历史列表内部滚动。
- 包含：移动端取消吸顶和相关自动化检查。

## 非目标

- 不包含：构建数据、路由、日志、轮询或 API 行为变化。
- 不包含：左右列宽和移动端内容顺序变化。

## 验收条件

- [ ] 宽屏历史栏吸顶且不会随右侧长日志滚出可视区域。
- [ ] 历史列表超出可用高度时在卡片内部滚动。
- [ ] 左侧卡片不被强制拉伸到右侧详情高度。
- [ ] 760px 及以下取消吸顶并维持现有上下布局。
- [ ] 前端测试、类型检查、生产构建和 `./scripts/verify.sh` 通过。
