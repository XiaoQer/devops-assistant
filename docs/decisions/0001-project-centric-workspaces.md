# ADR 0001：项目中心化工作空间

- 状态：已接受
- 日期：2026-07-02

## 背景

早期产品文档把 `Application` 视为平台顶层聚合。后续实现引入了项目、成员、集群、
项目级镜像仓库、项目导航和项目级开发者工作空间。

如果原样恢复旧的应用中心化描述，仓库文档会与当前代码和近期产品方向冲突。

## 决策

使用 `Project` 作为治理、归属和工作空间边界；使用 `Application` 作为项目内主要
交付聚合。

当项目影响范围或归属时，新的交付导航和 API 应携带项目上下文。应用级交付行为仍
围绕应用组织。

## 影响

- 产品和架构文档必须同时描述两个层级。
- 新功能必须明确是项目级还是应用级。
- 旧的全局路由可以暂时保留，但属于过渡状态。
- 后续权限体系可以自然地绑定到项目成员关系。
